🔬 MelanoCheck — System Architecture

End-to-end MLOps pipeline for low-resolution melanoma detection

Architecture Diagram

Block Explanations

Data Layer
Raw Data
32×32 pixel skin lesion images stored locally on filesystem.
  • 801 images tracked by DVC (local remote)
  • Two classes: malignant and benign
  • Git tracks .dvc pointer files for reproducibility
Airflow Pipeline
Apache Airflow 2.8.1 orchestrating 4-task data DAG.
  • validate: checks corrupt/zero-byte files
  • split: stratified 70/15/15 train/val/test
  • preprocess: resize 32×32 → 64×64 RGB
  • baseline_stats: pixel histogram for drift detection
Processed Data
64×64 RGB images + CSV manifests + baseline stats JSON.
  • 164 train / 36 val / 600 test images
  • baseline_stats/v1_stats.json for drift detection
Training Layer
MobileNetV3-Small
Pretrained on ImageNet, base layers frozen.
  • 1.5M total / 592K trainable parameters
  • Lightweight — fast on CPU
  • All runs tracked in MLflow experiment baseline-v1
EfficientNet-B0
Pretrained on ImageNet, base layers frozen.
  • 4M total / 2K trainable parameters
  • Stronger feature extractor than MobileNet
SimpleCNN
Custom CNN built from scratch — no pretrained weights.
  • 102K parameters, all trainable
  • 3 conv blocks + GAP + FC layers
  • Proves resolution is the bottleneck, not architecture
Model Registry
MLflow Registry
Local MLflow registry in mlruns/ folder.
  • Every run: params, per-epoch metrics, artifacts
  • Two-stage champion/challenger auto-promotion
  • Stage 1: compare val_recall (primary)
  • Stage 2: compare val_F1 (tiebreaker)
  • Production model auto-loaded by FastAPI
  • UI at http://localhost:5000
Serving Layer
FastAPI Backend
Python 3.10, FastAPI 0.111.0, port 8000.
  • /predict — classify image, save for retraining
  • /feedback — ground truth, update recall metrics
  • /health — liveness check
  • /ready — model readiness + auto-reload
  • /metrics — Prometheus scrape endpoint
nginx Frontend
nginx:alpine serving static HTML/CSS/JS, port 80.
  • Screening tab — upload and analyse images
  • Pipeline Dashboard tab — live metrics + tool links
  • API_BASE configurable via window.API_BASE
Docker Compose
5 services on shared mlops_network.
  • melanoma_frontend (port 80)
  • melanoma_backend (port 8000)
  • melanoma_mlflow (port 5000)
  • melanoma_prometheus (port 9090)
  • melanoma_grafana (port 3000)
Monitoring Layer
Prometheus
Scrapes /metrics every 15 seconds, port 9090.
  • 15 custom metrics tracked
  • Request counts, latency, predictions, errors
Grafana
NRT dashboard at http://localhost:3000, port 3000.
  • 13 panels: drift, recall, misclassification, etc.
  • Auto-provisioned from grafana/provisioning/
  • 3 alert rules: error rate >5%, drift detected, misclass >10%
Drift Detection
Runs on every /predict call.
  • Pixel histogram (10 bins) of incoming image
  • MAE vs baseline histogram from Airflow
  • Score > 20% → writes retrain_needed.flag
Retraining Layer
Retraining DAG
Airflow DAG running automatically every 30 minutes.
  • check_trigger → reads retrain_needed.flag
  • prepare_feedback_data → merge with training data
  • trigger_training → retrains all 3 models
  • evaluate_new_model → champion/challenger
  • cleanup → archive feedback images
Feedback Loop
Triggered when thresholds exceeded.
  • Misclassification rate > 10% (after 10+ feedback)
  • Drift score > 20%
  • Images saved with 0o777 permissions for cross-container access
Git + DVC
Full reproducibility stack.
  • Git: code, configs, .dvc pointer files
  • DVC: raw data (801 images, local remote)
  • dvc.yaml: 5-stage CI pipeline DAG
  • Every run: git hash + MLflow run ID
